Pages CMS setup — quick steps

1) Install Pages CMS GitHub App
- Visit https://app.pagescms.org/ and sign in with GitHub.
- Install the Pages CMS GitHub App on the account or organization that will host this repo.
- Grant the app access to the repository that contains this site.

2) Verify `.pages.yml` is present
- This repo already contains a `.pages.yml` at the repository root that maps:
  - `media.input` -> `images` (uploads will live under `/images`)
  - `content.articles` -> `content/*.md` (all article markdown files live in `/content`)
  - `content.settings` -> `content/settings.md`

3) Open the repo in Pages CMS
- From https://app.pagescms.org/ choose the installed repository. Pages CMS will read `.pages.yml` and expose collections.

4) Media & uploads
- Uploaded images will be written to the `images/` folder. Use relative paths like `images/your-image.jpg` in image fields.

5) Authentication & collaborators
- Pages CMS uses the GitHub App for commits; invite users via the Pages CMS UI (email invites) or add them as collaborators on the repository with write access.

6) Editing workflow
- Editors can create or edit markdown files from the UI; changes are committed to the repository and will trigger your usual CI/build (if configured).

7) Hooks & deploys (optional)
- If you have a build Action or deploy pipeline, add a workflow and (optionally) an `action` entry in `.pages.yml` to add an in-UI button.
- For GitHub Pages: ensure your deploy Action pushes the built static output to the branch configured for Pages (or use Pages’ branch).

8) Notes specific to this project
- Articles are plain Markdown files with YAML frontmatter. Fields expected: `title`, `main_image`, `header`, `hook`, `date`, `theme`, `body`.
- Media path is configured to `images`. There is already an `images/` folder in the repo.

9) Testing
- From Pages CMS, create a test article (it will create a new `.md` file in `content/`).
- Commit in the UI and verify `data/articles.json` gets regenerated by your build Action (if you rely on that) or run `npm run build:articles` locally.
